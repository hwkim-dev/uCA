# -- Core ---------------------------------------------------------------------
# Version pin rationale:
#   * sphinx<8    — myst-parser 5's hard coupling to a transform pulled from
#                   Sphinx core in 8.0 still blows up during setup, even
#                   though pip installs it cleanly. Revisit when myst-parser 6
#                   (Sphinx-8/9-ready) ships.
#   * myst-parser<5 — 5.0 calls `registry.transforms.remove()` unconditionally
#                   and cannot coexist with myst-nb 1.4 (which invokes the
#                   same setup path). 4.0.1 is the last working pairing.
#   * myst-nb<2   — keeps us on the 1.x API surface myst-parser 4 expects.
