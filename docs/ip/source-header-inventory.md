# Source-header inventory

Tracks per-repo coverage of the PCCX source-header policy described
in [`copyright-header-policy.md`](copyright-header-policy.md).

## Scope

The inventory counts files that have:

- a `PCCX(TM)` or `PCCX™` header line;
- an `SPDX-FileCopyrightText` line;
- an `SPDX-License-Identifier` line.

Excluded from the count:

- `.git/`, `node_modules/`, `target/`, `dist/`, `build/`, `out/`,
  `cache/`, `.venv/`, `venv/`, `__pycache__/`;
- `third_party/`, `vendor/`, submodule directories;
- `reference_to_cheet/`, `00_extracted_zips/` and similar local
  repository conventions used to hold extracted reference material;
- the `codes/` directory used by the `pccx` Sphinx build to clone
  external RTL at build time;
- `*.lock`, `package-lock.json`, `pnpm-lock.yaml`, `yarn.lock`,
  `Cargo.lock`, `go.sum`, `uv.lock`;
- `*.min.js`, `*.min.css`;
- files whose first 30 lines contain `generated`, `do not edit`, or
  `autogenerated`;
- binary files (detected by the presence of NUL bytes in the first
  2 KiB).

## Sources of truth

- [`copyright-header-policy.md`](copyright-header-policy.md) — what
  a header looks like and where it goes.
- This page — coverage reporting only. Coverage numbers are updated
  per rollout phase, not per commit.

## How coverage is reported

Each header rollout emits a JSON report (kept locally in the
operator's tooling workspace; not committed). The report contains:

- repo path
- total files visited
- files changed
- files skipped (with reason counts: `existing-spdx`,
  `generated`, `lockfile`, `minified`, `binary`,
  `ext-not-supported`, `not-file`)
- per-extension changed / skipped counts

Phase X rollout is the first systematic pass. Subsequent phases
should re-run the same script and update this inventory.

## Why this is not in `git`

The script can be regenerated from the policy and the file tree at
any time. Committing transient JSON reports adds noise to the
documentation history and would invite drift between the report and
the actual tree. Only the policy and the *current* coverage
narrative live in this repository.
