# Risk register

Engineering and adoption risks the project carries today. Bullet
entries; no estimates that are not already established. Items
marked TBD are real unknowns, not placeholders for confident
predictions.

## Engineering risks

- **Timing closure on KV260** — synthesis has been attempted; a
  completed timing-closed bitstream is pending. Until closure is
  reached, all on-board claims are TBD.
- **W4A8 golden-vector gate** — arithmetic RTL changes are gated
  by reproducible golden vectors before any behaviour change.
  Fixtures listed in
  `pccx-FPGA-NPU-LLM-kv260/docs/W4A8_GOLDEN_VECTOR_GATE.md` are
  TBD beyond the current smoke vectors.
- **Submodule pin drift after rebase merges** — known hazard; the
  [submodule pin policy](../reference/submodule-pin-policy.md)
  documents the repair procedure. The hazard remains for any
  future rebase-merged IP-core PR; mitigation is the procedure
  itself, not an automated guard yet.
- **Auto-translated KO documentation drift** — the EN tree is the
  source of truth; the KO tree is regenerated by external
  tooling. Drift between EN and KO is detected by the EN/KO file
  set CI gate but content drift inside matched files is not yet
  detected automatically.
- **Vivado toolchain version drift** — the IP-core is target-tool
  agnostic in spirit, but the board integration repo is currently
  validated against the specific Vivado version used for KV260
  bring-up. Newer Vivado versions are not yet exercised.

## Adoption risks

- **External cloners forgetting `--recurse-submodules`** — fresh
  clones without the flag look superficially complete but cannot
  build. The onboarding doc states the requirement; CI is not yet
  guarded against the failure mode.
- **CLA not yet published** — contributions to repos that may be
  used commercially are accepted under the existing open licence,
  but the CLA the project intends to ship is not yet finalised.
  See [contribution rules (draft)](../onboarding/contribution-rules.md).
- **Documentation lock-in to Sphinx** — tooling and CI assume the
  Sphinx + MyST + multilanguage layout. A future migration to a
  different documentation system would require coordinated rewrite
  across `pccx`, `pccx-v002`, and the board integration repo's
  `docs/`.

## What this page is not

- It is **not** a probability estimate. No likelihood ratings, no
  expected impact figures.
- It is **not** a mitigation plan. Mitigations live where the work
  lives (the relevant repo's CI, runbooks, or contracts).
- It is **not** stable. New risks are added as they are noticed;
  resolved risks are removed only when there is concrete evidence
  the risk no longer applies.
