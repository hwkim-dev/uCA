# Onboarding

Entry point for external reviewers and contributors. The onboarding
set is short on purpose — it points back to
[`docs/reference/`](../reference/README.md) for authoritative detail.

## Pages

- [Getting started](getting-started.md) — clone, toolchain
  expectations, how to run the Sail typecheck and the KV260 sim
  wrapper.
- [Architecture overview](architecture-overview.md) — short tour of
  the layers, with pointers into the reference set for everything
  else.
- [Contribution rules (draft)](contribution-rules.md) — what
  contributing this code does and does not entitle a contributor to.
  Marked as draft until the project's CLA is published.

## What this set is not

- It is **not** a roadmap. Roadmap items live under
  [`docs/roadmap/`](../roadmap/milestones.md).
- It is **not** an architecture spec. The spec lives under
  `docs/v002/` (and the future `docs/v003/`).
- It is **not** the source of truth for repository topology, the
  v002 contract, the boundary rule, or the submodule pin policy —
  those live under [`docs/reference/`](../reference/README.md).
